Your Custom Prompt

Hint: You can use placeholders in the format [id.action].
  • id: The ID of the requirement (e.g., R001).
  • action: One of the available actions to retrieve specific attributes of the requirement.

Example: [R001.formal] retrieves the formalizations of the requirement with ID R001.

Legend of Actions
Action Description
id The unique ID of the requirement.
desc The description of the requirement.
type The type of the requirement. If not provided as a string, None will be used.
tags A list of all tags associated with this requirement.
tags_comments A dictionary associating each tag with its comment.
formal A list of formalizations represented as strings.
vars An alphabetically sorted list of all variables used in the formalizations.
pos The position of the requirement in the CSV file.
status The current status of the requirement (e.g., active, deprecated).
csv_data The full CSV row describing the requirement.
type_inference_errors A list of errors encountered during type inference.
revision_diff Differences between the revisions of the requirement.
